## 5. Безопасность

### Аутентификация
- **HMAC-SHA256** верификация Telegram initData на каждом запросе
- Защита от replay-атак: окно 1 час
- Отклонение bot-аккаунтов (is_bot=true → 401)
- Автоматическое создание пользователя при первом запросе
- Блокировка забаненных пользователей на уровне middleware

### Авторизация
- **Пользователь**: Telegram initData (HMAC)
- **Супер-админ**: hardcoded SUPER_ADMIN_TG_ID (env override)
- **Оператор**: OPERATOR_SECRET заголовок (timing-safe сравнение)

### Транзакционная безопасность
- Все финансовые операции — атомарные процедуры PostgreSQL с `FOR UPDATE` блокировкой
- Гарантия: двойного списания невозможно

### Rate Limiting (по IP)
| Endpoint | Лимит |
|----------|-------|
| Общий | 120 req/мин |
| Чекин | 10 req/мин |
| Вывод | 5 req/час |
| Промо-клейм | 10 req/мин |

### Валидация входных данных
- Координаты: -90 ≤ lat ≤ 90, -180 ≤ lng ≤ 180
- Карта: только 16-значная последовательность, хранится маскированной
- Бюджет: ≥ 1 GEO, max_visits от 1 до 100,000
- Размер тела запроса: 64 KB

### Известные ограничения
- Сессии бота хранятся в памяти (теряются при рестарте)
- Приостановка (`suspended_at`) бизнеса не проверяется при чекине (косвенно защита через баланс)
- Нет IP-геофенсинга для супер-адмивского доступа
- Нет KYC/AML верификации пользователей

## 8. Деплой и мониторинг

### Railway (API + Bot)
- Единый процесс: `node start.js` запускает api + bot как дочерние процессы
- Auto-restart при падении (задержка 3 сек)
- Переменные окружения: BOT_TOKEN, DATABASE_URL, SERVICE_KEY, WEBAPP_URL, OPERATOR_SECRET, GEO_RATE

### Vercel (Mini App)
- Статический SPA (Vite build)
- Переменные: VITE_API_URL, VITE_APP_NAME

### Supabase
- Managed PostgreSQL
- Миграции — ручной запуск SQL в SQL Editor (нет миграционного фреймворка)

### Мониторинг
- Логи: Railway console (console.error / console.log)
- Структурированного логирования нет
- Alerting отсутствует

# GeoEarn — Продажное описание продукта
**Готовый к запуску Telegram Mini App | Дата: 2026-05-24**

## TL;DR

**GeoEarn** — полностью готовая платформа лояльности внутри Telegram. Пользователи получают GEO-токены за визиты в кафе, магазины и рестораны. Бизнесы получают измеримый трафик. Платформа зарабатывает комиссию. Всё работает без установки приложений.

**Запрашиваемая цена: от $35 000** (negotiable, зависит от условий)

## Ключевые функции

### Для пользователей

**Чекин за 3 секунды**
QR-сканирование → геоверификация на сервере → мгновенное начисление GEO. Пользователь видит баланс в реальном времени.

**GEO-кошелёк**
Полная история транзакций, баланс, статистика посещений. Вывод средств на карту (GEO → UZS) через форму в боте.

**Геймификация (полностью рабочая)**
- 10 уровней прогрессии (Новичок → Легенда) с множителями наград 1.0× – 1.25×
- Дневные стрики с бонусами на 7-й, 14-й и 30-й день (500 / 1 500 / 5 000 GEO)
- Заморозка стрика — 1 пропуск в неделю без сброса серии
- 12 заданий (ежедневные, еженедельные, разовые)
- 6 достижений за реальное поведение
- Буст-события с временным множителем до 1.5×

**Интерактивная карта**
Все партнёры на Leaflet-карте с геолокацией пользователя и расстоянием до каждой точки.

**Реферальная программа**
Уникальные ссылки, 1 000 GEO рефереру + 500 GEO новому пользователю, 5% пассивный доход от чекинов приглашённых (30 дней).

**Promo QR Hunt**
Отдельный режим охоты за промо-токенами с редкостью (common / rare / epic / legendary) и геолокационным ограничением.

**GeoHunt**
Географические квесты с кодами — пользователи находят коды в реальных локациях и сканируют их для получения наград.

### Для бизнеса

**Личный кабинет в Mini App** — не нужен отдельный продукт.

**Управление кампаниями**
- Создание кампании: бюджет → автоматический расчёт награды и лимита посещений
- PIN-верификация (6 цифр, 15 мин) — защита от фиктивных чекинов
- Уникальный QR-код на каждую кампанию
- Автоостановка при исчерпании бюджета

**Статистика**
Сегодня / 7 дней / предыдущие 7 дней. Количество посещений, расход бюджета, уникальные пользователи.

**Пополнение баланса**
Запрос через оператора, подтверждение с аудит-логом.

### Для платформы (SuperAdmin)

**Dashboard**
Общая выручка (platform_wallet), активные пользователи, объём выводов, статистика кампаний.

**Управление пользователями**

**Финансы**

- Управление курсом GEO → UZS с историей изменений

Подозрительная активность, мультичекины, анализ паттернов.

**Управление контентом**
Создание/редактирование промо-кампаний, заданий, достижений прямо из интерфейса.

## Как зарабатывает платформа

### Источник 1: Комиссия 10% с кампаний

```
Бизнес создаёт кампанию на 5 000 GEO
  ↓ Платформа атомарно получает 500 GEO
  ↓ Бизнес оплачивает → пользователи получают награды
  ↓ Комиссия записана в platform_wallet и platform_transactions
```

Комиссия реализована через PostgreSQL RPC-функцию `create_campaign_with_commission()` — деньги нельзя "потерять" между операциями.

### Источник 2: Спред при выводе

Платформа контролирует курс GEO → UZS через env-переменную `GEO_RATE`. Разница между себестоимостью и курсом — дополнительная маржа.

### Источник 3: Promo QR Hunt

Брендированные промо-кампании с платным размещением. Редкость токенов (legendary > epic > rare > common) позволяет дифференцировать тарифы.

### Юнит-экономика

| Сценарий | Бизнесов | Кампаний/мес | MRR | ARR |
|---------|---------|------------|-----|-----|
| Старт | 10 | 30 | $1 200 | $14 400 |
| Рост | 50 | 150 | $6 000 | $72 000 |
| Масштаб | 500 | 1 500 | $60 000 | $720 000 |

*Расчёт: средний бюджет кампании 5 000 GEO, курс 1 GEO = 1 000 UZS ≈ $0.08, комиссия 10%*

## Техническая архитектура

```
┌─────────────────────────────────────────────────────┐
│                   TELEGRAM                           │
│  ┌─────────────────┐    ┌──────────────────────┐    │
│  │   Telegram Bot   │    │  Mini App (SPA)       │    │
│  │   (Grammy)       │    │  React 18 + Vite      │    │
│  └────────┬────────┘    └──────────┬───────────┘    │
└───────────│──────────────────────────│───────────────┘
            │                          │ initData HMAC
            ▼                          ▼
┌───────────────────────────────────────────────────┐
│            Express API (Railway)                   │
│  validateTma.js — HMAC-SHA256 auth                 │
│  antifraud.js — rate limit + geo check             │
│  62 endpoints across 15 route files                │
└───────────────────────────┬───────────────────────┘
                            │
                            ▼
┌───────────────────────────────────────────────────┐
│           Supabase PostgreSQL                      │
│  20 tables + 10 atomic RPC functions               │
│  No race conditions on financial operations        │
└───────────────────────────────────────────────────┘
```

**Стек:**
- Backend: Node.js + Express
- Database: PostgreSQL (Supabase)
- Frontend: React 18 + Vite
- Bot: Grammy (Telegram Bot API)
- Hosting: Railway (API+Bot) + Vercel (Frontend)
- Auth: HMAC-SHA256 Telegram initData

## Безопасность

| Защита | Реализация |
|--------|-----------|
| Аутентификация | HMAC-SHA256 подпись initData, timing-safe сравнение |
| Replay-атаки | Проверка `auth_date` — не старше 24 часов |
| Race conditions | 10 PostgreSQL RPC с транзакционной изоляцией |
| Rate limiting | 10–120 req/min в зависимости от endpoint'а |
| Геофрод | Haversine проверка на сервере, клиент не доверяется |
| PIN-верификация | 6 цифр, TTL 15 минут на каждый PIN |
| Мультичекин | 24-часовое ограничение на пару user+business |
| Секреты | BOT_TOKEN, SERVICE_KEY, OPERATOR_SECRET в env-переменных |

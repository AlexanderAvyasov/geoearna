# GeoEarn — Полный гайд по работе системы

---

## 1. Что такое GeoEarn и как это работает в целом

GeoEarn — это Telegram Mini App, в которой пользователи зарабатывают **GEO-монеты** за посещение заведений (кафе, магазины, рестораны и т.д.), а потом выводят их в реальные деньги (UZS) через Payme.

### Кто участвует в системе

| Роль | Кто | Что делает |
|------|-----|-----------|
| **Пользователь** | Обычный человек в Telegram | Сканирует QR, зарабатывает GEO, выводит деньги |
| **Бизнес** | Владелец заведения | Пополняет баланс GEO, создаёт кампании, размещает QR |
| **Суперадмин** | Ты (Telegram ID 930826522) | Управляет всем: подтверждает пополнения, обрабатывает выводы, видит статистику |

### Как деньги движутся через систему

```
Бизнес платит деньги (UZS)
        ↓
Суперадмин подтверждает → бизнесу начисляется GEO-баланс
        ↓
Бизнес создаёт кампанию (тратит GEO из своего баланса)
        ↓
Пользователи сканируют QR → получают GEO
        ↓
Пользователи запрашивают вывод GEO → UZS
        ↓
Суперадмин вручную отправляет деньги на Payme пользователя
```

### Курс GEO

По умолчанию **1 GEO = 600 UZS**. Меняется в суперадминке (вкладка "Система").

## 2. Как добавить новый бизнес (пошагово)

Бизнесы создаются вручную в **Supabase** — через веб-интерфейс базы данных.

### Шаг 1. Зайди в Supabase

- Открой [supabase.com](https://supabase.com) → твой проект
- Перейди в **Table Editor** → таблица `businesses`

### Шаг 2. Добавь строку в таблицу `businesses`

Нажми **+ Insert row** и заполни:

| Поле | Что писать | Пример |
|------|-----------|--------|
| `name` | Название заведения | `Кафе Весна` |
| `address` | Адрес | `ул. Навои 12, Ташкент` |
| `owner_telegram_id` | Telegram ID владельца | `123456789` |
| `balance` | Начальный GEO-баланс | `0` |
| `lat` | Широта заведения | `41.2995` |
| `lng` | Долгота заведения | `69.2401` |
| `qr_token` | Уникальный токен для QR | сгенерируй случайную строку, напр. `abc123xyz` |

> **Как узнать Telegram ID** владельца: попроси его написать боту `/start` или используй сайт [@userinfobot](https://t.me/userinfobot)

> **Координаты** можно взять из Google Maps: правая кнопка на точке → "Что здесь?" — скопируй цифры.

> **qr_token** — любая уникальная строка без пробелов, 8–30 символов. Например: `vesna-cafe-001`

### Шаг 3. QR-код для заведения

QR-код ведёт на ссылку формата:
```
https://ВАШ_WEBAPP_URL/checkin?token=ТОКЕН_ЗАВЕДЕНИЯ
```

Сгенерировать QR можно через любой онлайн-генератор или прямо из **бизнес-панели** (кнопка "Скачать QR" в карточке кампании).

### Шаг 4. Сообщи владельцу бизнеса

После создания строки в Supabase владелец открывает приложение → вкладка "Бизнес" → видит свою панель управления.

## 3. Как работает панель бизнеса

Владелец заведения видит вкладку **"Бизнес"** (иконка магазина в нижней навигации).

### Вкладка "Обзор"

- Визиты за сегодня, за 7 дней, прирост в % к прошлой неделе
- GEO-баланс заведения
- Кнопка **"Сгенерировать PIN"** — создаёт одноразовый PIN на 15 минут для чекинов с защитой

### Вкладка "Кампании"

Кампания = акция, за которую пользователи получают GEO.

**Как создать кампанию:**

1. Нажать **"+ Создать кампанию"**
2. Заполнить:
   - **Бюджет** (в GEO) — сколько GEO потратит заведение на эту акцию
   - **Количество визитов** — максимум сколько раз можно получить награду
   - **Тип задания**: Визит / Покупка / Отзыв
   - **Описание** (опционально) — что нужно сделать пользователю
   - **Требуется PIN?** — включить, если сотрудник должен подтвердить присутствие
   - **Дата окончания** (опционально)
3. Нажать **"Запустить"**

**Расчёт награды пользователю:**
```
Награда за визит = (Бюджет × 90%) ÷ Количество визитов
```
Пример: бюджет 10 000 GEO, 100 визитов → каждый пользователь получит **90 GEO**

10% от бюджета идут как комиссия платформы (поступают на счёт Platform Wallet).

**Что показывает карточка кампании:**
- Статус: Активна / Остановлена / Завершена
- Прогресс-бар (сколько визитов использовано из максимума)
- Дата окончания
- Кнопка "Скачать QR" — скачивает PNG-файл с QR-кодом
- Кнопка "Стоп" — останавливает кампанию досрочно

### Вкладка "Пополнение"

Бизнес пополняет свой GEO-баланс через перевод денег на карту/Payme платформы.

**Пакеты пополнения:**
| Пакет | GEO |
|-------|-----|
| Старт | 10 000 GEO |
| Рост | 50 000 GEO |
| Бизнес | 100 000 GEO |

**Процесс:**
1. Бизнес выбирает пакет → видит реквизиты карты (из `.env`: `TOPUP_CARD_NUMBER`)
2. Переводит деньги вручную
3. В приложении нажимает **"Я оплатил"** → заявка создаётся в статусе "Ожидает"
4. **Суперадмин подтверждает** в своей панели → GEO зачисляются бизнесу

> Реквизиты карты для приёма платежей настраиваются в `.env` через `TOPUP_CARD_NUMBER`, `TOPUP_CARD_HOLDER`, `TOPUP_BANK`

## 4. Как работает Суперадминка

Доступна только для Telegram ID `930826522`. Открывается через нижнюю навигацию (иконка щита).

### Вкладка "Обзор"

Общая статистика платформы:
- Всего пользователей / визитов / бизнесов / активных кампаний
- Баланс Platform Wallet (комиссии)
- Сколько GEO выдано всего
- Сколько ожидает вывода (количество и сумма в GEO)

### Вкладка "Выводы"

Здесь все заявки пользователей на вывод GEO → UZS.

**Статусы:**
- **Ожидает** — нужно обработать вручную
- **Одобрено** — деньги отправлены
- **Отклонено** — заявка отменена, GEO возвращены пользователю

**Как одобрить вывод:**
1. Нажать на строку с заявкой
2. В открывшейся карточке видно: сумма GEO, сумма UZS, номер Payme
3. **Физически отправить деньги** на указанный Payme-кошелёк
4. Нажать кнопку **"Одобрить"**
5. Пользователю автоматически придёт уведомление в Telegram

**Как отклонить:**
1. Нажать на строку
2. Нажать **"Отклонить"** → написать причину (опционально)
3. GEO автоматически возвращаются пользователю
4. Пользователю придёт уведомление

### Вкладка "Пользователи"

Список всех пользователей, отсортированный по балансу.

Действия над каждым пользователем (нажать на строку):
- **Ручная корректировка GEO** — добавить или убрать GEO (можно отрицательное число)
- **Заблокировать** — пользователь не сможет делать чекины
- **Приостановить** — временная блокировка

### Вкладка "Бизнесы"

Список всех заведений.

Действия:
- Просмотр баланса GEO каждого бизнеса
- **Подтвердить пополнение** — нажать на заявку, нажать "Подтвердить" → GEO зачислятся бизнесу

### Вкладка "Система"

- **Курс GEO** — изменить, сколько UZS стоит 1 GEO (применяется ко всем выводам)
- Другие настройки платформы

### Вкладка "Геймиф." (Геймификация)

Управление заданиями и достижениями:
- Редактировать название, награду в GEO, награду в XP любого задания
- Добавлять / удалять задания и достижения

> Изменения применяются к новым прогрессам. Уже выполненные задания не пересчитываются.

## 5. Как работает чекин (процесс для пользователя)

1. Пользователь открывает приложение
2. Нажимает **центральную кнопку сканера** в нижней навигации
3. Наводит камеру на QR-код заведения
4. Приложение определяет геолокацию — проверяет, что пользователь физически находится рядом с заведением
5. Если кампания требует PIN — пользователь просит сотрудника назвать PIN
6. GEO зачисляются мгновенно
7. Задания и достижения обновляются автоматически

**Почему чекин может не пройти:**
- Пользователь слишком далеко от заведения (`TOO_FAR`)
- Уже чекинился здесь сегодня (`TOO_SOON`)
- Нет активных кампаний (`NO_ACTIVE_CAMPAIGN`)
- Неверный или просроченный PIN (`INVALID_PIN`)
- Кампания исчерпана (все визиты использованы)

## 6. Геймификация — задания, достижения, уровни

### Задания

Три типа:
- **Ежедневные** — сбрасываются каждый день в 00:00 по Ташкенту
- **Еженедельные** — сбрасываются каждый понедельник
- **Разовые** — выполняются один раз за всё время

Когда задание выполнено — в приложении появляется кнопка **"Забрать"**. GEO и XP зачисляются только после нажатия.

### Достижения

Выдаются автоматически при достижении условия (без кнопки "забрать"). GEO и XP начисляются сразу.

### XP и уровни

| Уровень | Название | XP от | XP до |
|---------|----------|-------|-------|
| 1 | Новичок | 0 | 100 |
| 2 | Исследователь | 100 | 250 |
| 3 | Постоянный | 250 | 500 |
| 4 | Активный | 500 | 1 000 |
| 5 | Эксперт | 1 000 | 2 000 |
| 6 | Мастер | 2 000 | 3 000 |
| 7 | Ветеран | 3 000 | 3 750 |
| 8 | Элита | 3 750 | 4 500 |
| 9 | Чемпион | 4 500 | 5 000 |
| 10 | Легенда | 5 000 | — |

Более высокий уровень даёт бонус к наградам за чекин (умножитель от 1.00 до 1.25).

### Стрик

Ежедневная серия чекинов. Юбилейные дни (7, 14, 30) дают бонус ×1.5 к следующему чекину.
Стрик замораживается автоматически если пропустить день — число заморозок ограничено.

### Рефералы

- Пользователь получает уникальную ссылку (в разделе "Прогресс" → "Рефералы")
- Если приглашённый друг делает первый чекин — оба получают бонус:
  - Пригласивший: **+1 000 GEO** + **5% от каждого чекина друга в течение 30 дней**
  - Новый пользователь: **+500 GEO** приветственный бонус

## 9. Типичные задачи — быстрая шпаргалка

### Добавить бизнес
→ Supabase → Table Editor → `businesses` → Insert row

### Пополнить баланс бизнеса вручную (без заявки)
→ Supabase → `businesses` → найти строку → изменить `balance`

### Подтвердить пополнение от бизнеса
→ Суперадминка → вкладка "Бизнесы" → нажать на заявку → Подтвердить

### Одобрить вывод пользователя
→ Суперадминка → вкладка "Выводы" → нажать на строку → отправить деньги на Payme → Одобрить

### Изменить курс GEO
→ Суперадминка → вкладка "Система" → ввести новый курс

### Дать/убрать GEO у пользователя
→ Суперадминка → вкладка "Пользователи" → нажать на пользователя → Ручная корректировка

### Заблокировать пользователя
→ Суперадминка → вкладка "Пользователи" → нажать на пользователя → Заблокировать

### Изменить награды за задания
→ Суперадминка → вкладка "Геймиф." → нажать на задание → изменить значения

### Обновить значения заданий в базе данных
→ Supabase → SQL Editor → вставить и выполнить блок из `db/schema.sql` (строки с `INSERT INTO task_definitions`)

### Посмотреть все pending выводы в БД напрямую
```sql
SELECT id, amount, phone, created_at FROM withdrawals WHERE status = 'pending' ORDER BY created_at;
```

## 10. Важные ограничения системы

- **Минимальный вывод**: 50 000 UZS (эквивалент GEO по текущему курсу)
- **Комиссия платформы**: 10% от бюджета каждой кампании
- **Один чекин в сутки** на одно заведение (блокировка `TOO_SOON`)
- **Геолокация обязательна** — чекин не пройдёт без разрешения на геолокацию
- **PIN действует 15 минут** с момента генерации, одноразовый
- **Суперадмин**: только один, Telegram ID захардкожен в `api/routes/superadmin.js` (строка 9)
- **Рефералы**: друг должен сделать первый чекин в течение 7 дней после регистрации по ссылке
